Project Source Files

The learning objectives of this project was to execute data analysis activities in a group setting effectively by practicing collaborative techniques including the use of a repository to store and share project assets as they were being developed by individual members. The team implemented the GitHub repository "projWANGRY", where they stored reference materials, python scripts, extracted and generated data files, generated graph images, and the final presentation.

The following is the directory structure of the repository:

  • 0_Reference - Reference materials including articles and images from the internet
  • 1_Input - Data files manually downloaded or extracted via API from identified public data sources
  • 2_LoadCode - Development area for Python scripts that would load code from web APIs or raw data files
  • 3_ManipulationCode - Development area for Python scripts that would clean up the data and merge data sets
  • 4_OutputCode - Development area for Python scripts that would graph merged data sets and output as images
  • 5_FinalCode - All properly working Python scripts to be evaluated by the teaching assistants
  • 6_Output - All graph images and output created by the Python scripts

Note: The parent folder holds all presentation documentation including the proposal and PowerPoint files.

This repository was originally intended to provide organization during project execution for the purpose to be intiutive for new eyes to reproduce the analysis. Unfortunately in practice with tight schedules, and unfamiliarity with processes for operating in a shared environment, it was difficult to adhere to the intentions of this structure. Perhaps later I will go through and properly clean up the repository to a state I had originally envisioned.
