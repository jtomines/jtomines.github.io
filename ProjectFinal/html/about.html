About: The Process of Visualization

Project Information

Course: University of Toronto SCS Data Analytics Boot Camp
Assignment: Final Project
Author: Jose Tomines
Date: 2019 April 18

This final project was meant to demonstrate the student's knowledge and abilities acquired throughout this course to create something meaningful. Many possible projects were mulled over, but this one held meaning in terms of content, and was still deliverable within a reasonable timeframe.

Project Methodology

The following were the tasks performed during this project as set out in the class proposal:

  • Determine data sources
  • Determine disualization of data
  • Determine framework of website
  • Build data extraction and cleanup scripts
  • Build database
  • Build visualization code
  • Analyze and comment on visualization
  • Assemble website

Project Execution Notes

The following are comments, notes, and issues encountered during project executions:

  • Most data sources provided a means to download data as either an excel or csv file. Where this option was not available, the data was copied directly from the page and pasted into an excel spreadsheet.
  • All mass shooting data from different sources were brought together in the same spreadsheet, and manually inspected to remove duplicate incidences.
  • All data sources in excel format were then manipulated using Excel and VBA functionality and then saved as csv files to be used by the Python scripts.
  • Python scripts using pandas and numpy imported the data from the csv viles, to further clean up and combine the data into dataframe tables to be used by the visualization scripts.
  • The original plan was to build a SQL database and serve the data with Flask to the front end website. Unfortunately time constraints led to the decision to keep the data as csv files and have the javascript files pull the data in from the csv files. Eventually this website will use an SQL database and Flask engine.
  • The website is html/css using Bootstrap 4.2.1, with javascript using jquery 3.3.1. The visualization used D3 and heavily modeled after Mike Bostock's chloropleth code
  • Python with pandas, numpy, and matplotlib, was used to create the static graphs as png files.
  • Chloropleth map projects were not dynamically sized and positioned, leading to placement issues for larger browser windows. This will be fixed in the future.
  • Dot projections on chloropleth mapping relies heavily on longitude and latitude. For state aggregated data, the longitude and latitude of the geographic center of the state (centroid methodology) was used, as described in Wikipedia - List of geographic centers of the United States (Updated list of geographic centers).
  • Normalization of state aggregated data (per 100K persons or per 1M persons) used population estimates for 2018 in United States Census Bureau - National Population Totals and Components of Change: 2010-2018.
